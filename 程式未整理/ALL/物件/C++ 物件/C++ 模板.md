# C++ 模板
***
>[!info]+
>模板能作用在class與func上，宣告與定義都要加上

>模板類別的定義與實體化通常放在<u>同一個源文件</u>中，若定義與實體分開連結時會出問題

可參考：
[[C++ 多文件模板LNK1120 2019]]

***

## 模板類的繼承
>[!info]+
>模板在繼承中可能會造成 `C3861` `C2065` 未宣告的識別項
>因為在繼承的時後<mark style="background: #FFB86CA6;">父類模板未實體化</mark>，定義時會有問題

- 此時可以在子類使用 this指標->變數 或 `類名<T>`::變數 

詳細資料：[[C++ 模板類的繼承]]
